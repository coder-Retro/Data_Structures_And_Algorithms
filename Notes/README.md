# 📝 Notes

> **"Code shows you what works. Notes remind you why."**

This section holds handwritten/scanned study notes covering the theory behind the data structures and algorithms implemented elsewhere in this repo. While the code folders focus on *implementation*, these notes focus on *intuition* — diagrams, definitions, and step-by-step reasoning captured while first learning each topic.

Think of this as the textbook companion to the rest of the repository: read the note before touching the code, then use the code to verify the concept actually clicked.

---

# 🎯 Learning Objectives

By reviewing this section, you should be able to:

* Build a conceptual foundation before jumping into implementation.
* Revisit core definitions (e.g. what makes a stack LIFO, a queue FIFO) without re-deriving them from code.
* Understand *why* certain data structures exist before learning *how* to code them.
* Use complexity notes as a quick refresher before analyzing TC/SC in LeetCode solutions.

# 💡 How to Use These Notes

1. Read the note for a topic **before** opening its corresponding folder in `Data_Structures/` or `STLContainers/`.
2. Try to explain the concept out loud (or in writing) after reading — if you can't, revisit the note.
3. Only move to the implementation once the *why* behind the structure makes sense.
4. Come back to note 5 (Time And Space Complexity) whenever a TC/SC comment in `LeetCode/` feels unclear.
